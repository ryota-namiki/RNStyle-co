# 活用事例

cc-company は使う人によって育ち方が変わります。3つの具体的なシナリオを紹介します。

## 1. フリーランスWeb開発者

**プロフィール:** 受託開発メイン。複数クライアントを並行で抱えている。

### セットアップ

```
秘書: まず、事業や活動を教えてください。
あなた: フリーランスのWeb開発。受託が中心です

秘書: 今の目標や困りごとは？
あなた: 案件が増えてきてタスク管理が追いつかない。請求漏れも怖い
```

### 1ヶ月目：秘書だけで運用

```
> 今日やること
秘書: 今日のTODOです:
  - [ ] A社：トップページのレビュー対応 | 優先度: 高
  - [ ] B社：見積もり作成
  - [ ] 確定申告の領収書整理

> A社からSlackで修正依頼きた、対応する
秘書: TODOに追加しました！
  - [ ] A社：Slack修正依頼対応 | 優先度: 高
```

### 2ヶ月目：営業・経理が追加

```
秘書: クライアント関連のタスクが増えていますね。
      営業部門を作りましょうか？案件やクライアント情報を整理できます。
あなた: いいね、作って

秘書: 請求書の管理も何度か出てきました。
      経理部門も作りましょうか？
あなた: それも作って
```

### 最終的な組織

```
.company/
├── secretary/    ← 日々のTODO・メモ
├── sales/        ← クライアント情報・案件管理
│   ├── clients/
│   │   ├── company-a.md
│   │   └── company-b.md
│   └── proposals/
└── finance/      ← 請求書・経費
    ├── invoices/
    └── expenses/
```

## 2. 個人開発者（SaaS立ち上げ）

**プロフィール:** 会社員。副業で SaaS を開発中。まだローンチ前。

### セットアップ

```
秘書: まず、事業や活動を教えてください。
あなた: 副業で個人開発してます。タスク管理SaaSを作ってる

秘書: 今の目標や困りごとは？
あなた: 3ヶ月以内にローンチしたい。やることが多すぎて優先順位がつかない
```

### 1ヶ月目：秘書でアイデア整理

```
> 認証はSupabase AuthかClerk、どっちがいいかな
秘書: 壁打ちしましょう！それぞれの特徴を整理しますね...
      ...
      結論：初期はSupabase Authでシンプルに。スケール時にClerk移行も可能。
      メモに保存しておきますね。

> あ、料金ページにFAQ追加したいんだった
秘書: inboxに入れておきますね！
```

### 2ヶ月目：PM・開発が追加

```
秘書: 開発タスクのメモが増えてきましたね。
      PM部門を作って、マイルストーンとチケットで管理しましょうか？
あなた: お願い

秘書: 技術的な設計メモも溜まってきました。
      開発部門にまとめましょうか？
あなた: それもいいね
```

### 3ヶ月目：ローンチに向けてマーケティング追加

```
> そろそろLPとか告知の準備しないと
秘書: マーケティング部門を作りましょうか？
      LP、SNS告知、ブログ記事の管理ができます。
あなた: 作って！
```

### 最終的な組織

```
.company/
├── secretary/      ← アイデア・日々のメモ
├── pm/             ← ローンチまでのマイルストーン
│   ├── projects/
│   │   └── saas-launch.md
│   └── tickets/
├── engineering/    ← 技術設計・デバッグログ
│   ├── docs/
│   │   ├── auth-design.md
│   │   └── db-schema.md
│   └── debug-log/
└── marketing/      ← LP・告知計画
    ├── content-plan/
    │   ├── landing-page.md
    │   └── launch-blog-post.md
    └── campaigns/
        └── launch-campaign.md
```

## 3. 動画クリエイター

**プロフィール:** YouTube をメインに活動。企業案件も受けている。

### セットアップ

```
秘書: まず、事業や活動を教えてください。
あなた: YouTubeで動画作ってます。企業案件もやってる

秘書: 今の目標や困りごとは？
あなた: 投稿が不定期になりがち。企画のネタ切れも悩み
```

### 1ヶ月目：秘書で企画管理

```
> 動画のネタ思いついた。「初心者向けPremiere Pro講座」
秘書: いいですね！inboxに入れておきます。

> 来週の投稿どうしよう
秘書: inboxにストックしたネタを見てみましょう:
  - 初心者向けPremiere Pro講座
  - Vlogの編集テクニック5選
  - After Effects vs Motion Graphics比較
  どれにしますか？
```

### 2ヶ月目：マーケティングが追加

```
秘書: コンテンツ企画の相談が増えていますね。
      マーケティング部門で投稿スケジュールを管理しましょうか？
あなた: あー、それ欲しかった！
```

### 3ヶ月目：営業が追加（企業案件が増えてきた）

```
> D社から案件の問い合わせきた
秘書: いいですね！企業案件の管理も増えてきたので、
      営業部門を作りましょうか？提案書やクライアント管理ができます。
あなた: 作って
```

### 最終的な組織

```
.company/
├── secretary/      ← ネタ帳・日々のメモ
│   ├── inbox/      ← 思いついたネタをストック
│   ├── todos/      ← 撮影・編集のTODO
│   └── notes/      ← 壁打ちメモ
├── marketing/      ← 投稿スケジュール・企画管理
│   ├── content-plan/
│   │   ├── premiere-pro-tutorial.md
│   │   └── vlog-editing-tips.md
│   └── campaigns/
└── sales/          ← 企業案件の管理
    ├── clients/
    │   └── company-d.md
    └── proposals/
```

## 共通のポイント

どのケースでも共通しているのは：

1. **最初は秘書だけ** — 気軽に始められる
2. **使っていくうちに部署が増える** — 無理に最初から決めない
3. **人によって組織が違う** — 同じプラグインでも育ち方が変わる
4. **秘書に話しかけるだけ** — 部署の存在を意識しなくていい
