# 部署を追加する

cc-company の核心機能です。使っていくうちに、秘書が部署の追加を提案します。

## 秘書からの提案

同じ領域のタスクが繰り返されると、秘書が提案してきます。

```
> 競合サービスについて調べて
秘書: 承知しました。調べますね。
  → secretary/notes/2026-03-16-competitor-research.md に保存

（数日後...）

> 海外のトレンドも調べたい
秘書: リサーチの依頼が増えていますね。
      リサーチ部門を作りましょうか？
      専用フォルダで調査結果を体系的に管理できます。

あなた: 作って

→ research/ が自動生成される
```

::: tip ポイント
部署がまだない場合、秘書は `secretary/notes/` に結果を保存します。部署ができたら、そちらに専用管理されるようになります。
:::

## 自分から依頼する

もちろん、直接依頼もできます。

```
> PM部門を作って
秘書: PM部門を作成しますね！

→ pm/ が自動生成される
```

## 追加される部署の構成

部署が追加されると、以下が自動生成されます：

- **部署フォルダ** — `.company/[department]/`
- **部署の CLAUDE.md** — その部署のルールと振る舞い
- **サブフォルダ** — 部署ごとのファイル管理用
- **テンプレート** — ファイル作成時の雛形

例えば、リサーチ部門を追加すると：

```
.company/
├── CLAUDE.md              ← 部署一覧が更新される
├── secretary/
└── research/              ← 新しく追加
    ├── CLAUDE.md           ← リサーチのルール
    └── topics/             ← 調査トピック管理
```

## 利用可能な部署

| 部署 | 担当領域 |
|------|---------|
| PM | プロジェクト進捗、チケット管理 |
| リサーチ | 市場調査、競合分析、技術調査 |
| マーケティング | コンテンツ企画、SNS、キャンペーン |
| 開発 | 技術ドキュメント、設計、デバッグ |
| 経理 | 請求書、経費、売上管理 |
| 営業 | クライアント管理、提案書 |
| クリエイティブ | デザインブリーフ、ブランド管理 |
| 人事 | 採用管理、チーム管理 |

詳細は [部署一覧リファレンス](/reference/departments) を参照してください。
