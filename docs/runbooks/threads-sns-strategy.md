# Threads 受託獲得SNS戦略

## 目的
UIUXデザイン・WEB制作の受託案件を増やす。
投稿を見た人が「この人に頼みたい」と思う状態を作る。

## ターゲット
- 中小企業・スタートアップのオーナー・マーケ担当
- 「サイトはあるけど問い合わせが来ない」という課題を持つ人
- リニューアルを検討しているが何から始めるか迷っている人

## 投稿の5本柱

| 柱 | 割合 | 目的 |
|---|---|---|
| 痛みを言語化 | 30% | 共感・リーチ拡大 |
| Before/After・事例 | 25% | 実力証明 |
| プロセス公開 | 20% | 信頼・差別化 |
| 数字・結果 | 15% | 信頼証明 |
| 人柄・考え方 | 10% | 親近感 |

## 投稿テンプレート集

### 【痛み系】
```
「〇〇なのに問い合わせが来ない方へ」

よくある原因はこれです：
・△△
・△△
・△△

実はUIの問題じゃなくて、
情報設計の問題がほとんど。

---
UIUXデザイン×WEB制作でご相談受付中
プロフのリンクからどうぞ
```

### 【Before/After系】
```
「リニューアル前後、こう変わりました」

Before：〇〇が課題
→ After：△△を解決

ポイントは「ユーザーが迷わない導線設計」

詳細はプロフのポートフォリオから
```

### 【プロセス公開系】
```
「ヒアリングで必ず聞く3つの質問」

①「今のサイトで一番困っていることは？」
②「理想のお客さんはどんな人ですか？」
③「競合と比べた自社の強みは何ですか？」

デザインの前に、ビジネスを理解する。
それが成果につながるWEB制作だと思ってます。
```

### 【数字・結果系】
```
「フォームの改善だけでCVRが変わった話」

やったこと：
・入力項目を8→4に削減
・エラーメッセージをわかりやすく
・送信ボタンのコピーを変更

小さな改善でも、ユーザー体験は大きく変わります。
```

### 【人柄系】
```
「なぜUIUXデザインにこだわるのか」

見た目がきれいなサイトより、
使いやすくて目的を達成できるサイトの方が
ビジネスの成果につながると信じているから。

デザインは手段、ゴールは依頼主の成功。
```

## 投稿スケジュール

- 頻度: 週3〜5投稿
- 曜日: 火・木・土（+α）
- 時間: 12:00 or 20:00（エンゲージメント高い帯）
